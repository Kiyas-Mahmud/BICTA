# BICTA — Annual Competition Website

Public website + custom admin panel for the yearly **BICTA** tech competition (Project Showcase, Datathon, Hackathon, …). Every event, competition, prize, news article, sponsor, judge, winner, FAQ, gallery photo and home-page section is editable from the admin UI — **no code changes between yearly editions**. Participants register through a built-in form; organizers review and export registrations and newsletter subscribers.

**Stack:** Nuxt 4 + TypeScript · Tailwind CSS · Drizzle ORM · SQLite · nuxt-auth-utils · Zod · Nuxt Icon (Lucide)

Plan / design docs live in [`docs/`](docs/): Project, Architecture, Security, Implementation, PRODUCT, DESIGN.

## 1. Prerequisites

| Tool | Version | Notes |

| **Node.js** | 20 LTS or 22 LTS | Tested on v22.15. Ships with npm. |

No database server to install — the app uses an embedded **SQLite** file. Everything runs locally with zero external services.

## 2. Quick start (any machine)

```bash
git clone <your-repo-url> bicta
cd bicta

cp .env.example .env          # macOS/Linux. Windows PowerShell: copy .env.example .env
#  → open .env and set the values (see §3)

npm run setup                 # installs deps, creates the DB, seeds admin + sample data
npm run dev                   # http://localhost:3000
```

`npm run setup` is shorthand for `npm install && npm run db:migrate && npm run seed`. Run those three individually if you prefer.

- **Public site:** http://localhost:3000
- **Admin panel:** http://localhost:3000/admin — log in with the `ADMIN_EMAIL` / `ADMIN_PASSWORD` from your `.env`.

## 3. Environment variables (`.env`)

Copy `.env.example` → `.env` and fill these in. `.env` is gitignored — never commit it.

| Variable | Required | Purpose |
|---|---|---|
| `NUXT_SESSION_PASSWORD` | ✅ | Secret key that encrypts the admin session cookie. **Must be ≥ 32 characters.** |
| `ADMIN_NAME` | ✅ | Display name for the first admin. |
| `ADMIN_EMAIL` | ✅ | Login email for the first admin (created by `npm run seed`). |
| `ADMIN_PASSWORD` | ✅ | Login password. **Min 12 chars.** The seed refuses weak/default values when `NODE_ENV=production`. |
| `DATABASE_URL` | ❌ | Reserved for a future PostgreSQL backend (see §5). **Leave unset** — the app currently always uses SQLite. |

**Generate a strong session password** (works on every OS, needs only Node):

```bash
node -e "console.log(require('crypto').randomBytes(24).toString('hex'))"
```

Paste the output as `NUXT_SESSION_PASSWORD`.

Minimal working `.env`:

```ini
NUXT_SESSION_PASSWORD=paste-the-64-char-hex-string-here
ADMIN_NAME=Admin
ADMIN_EMAIL=admin@bicta.local
ADMIN_PASSWORD=change-me-min-12-chars
```

## 4. The database (SQLite)

There is **nothing to install or connect to**. Drizzle ORM talks to a single SQLite file:

- **Location:** `.data/bicta.db` (created automatically; the `.data/` folder is gitignored).
- **Schema:** defined in [`server/database/schema.ts`](server/database/schema.ts) — the single source of truth.
- **Connection:** [`server/database/client.ts`](server/database/client.ts) — the only file that knows which driver is used.
- **Migrations:** SQL files in `server/database/migrations/`, generated by Drizzle Kit and committed to the repo.

### Commands

```bash
npm run db:migrate     # apply committed migrations (creates .data/bicta.db if missing)
npm run seed           # create the admin user from .env + sample content (dev only)
npm run db:generate    # ONLY after editing schema.ts — generates a new migration file
```

### Reset the database (start fresh)

Delete the file and re-run migrate + seed:

```bash
# macOS/Linux
rm -f .data/bicta.db* && npm run db:migrate && npm run seed

# Windows PowerShell
Remove-Item .data\bicta.db* -Force -ErrorAction SilentlyContinue; npm run db:migrate; npm run seed
```

`npm run seed` only inserts sample events/competitions/etc. on an empty database; it always upserts the admin account so you can use it to (re)set admin credentials.

### Moving data between machines

The whole database is one file. To copy your content to another machine, just copy **`.data/bicta.db`** (and the **`public/uploads/`** folder for images) over after installing dependencies. No dump/restore needed.

## 5. Using PostgreSQL instead (optional, future)

SQLite is perfect for development and small/medium production traffic. To move to Postgres later:

1. Add a Postgres `DATABASE_URL` to `.env`.
2. Implement the driver switch in [`server/database/client.ts`](server/database/client.ts) (use `drizzle-orm/node-postgres` when `DATABASE_URL` is set) and set `dialect: 'postgresql'` in `drizzle.config.ts`.
3. Regenerate + run migrations against Postgres.

This is intentionally isolated to one file so the rest of the app is unaffected. It is **not wired up yet** — out of the box the app is SQLite-only.

## 6. Running on another local machine — checklist

1. Install **Node 20 or 22** (`node -v` to confirm).
2. `git clone <repo>` and `cd` into it.
3. Create `.env` from `.env.example`; set `NUXT_SESSION_PASSWORD` (≥32 chars) and admin credentials.
4. `npm run setup` (install + migrate + seed).
5. `npm run dev` → open http://localhost:3000.
6. Log in at `/admin` with your `.env` admin credentials.

That is the entire setup. If `npm run dev` reports the port is busy, another process holds 3000 — stop it, or run `npm run dev -- --port 3001`.


## 9. Project structure

```
app/                       # Nuxt app (frontend)
  assets/css/main.css      # design system: tokens, buttons, cards, animations
  components/site/         # public UI (Hero parts, cards, Timeline, marquee, etc.)
  components/admin/        # admin UI (Collection editor, RichText, ImageUploader…)
  layouts/                 # default (public) + admin shells
  middleware/admin.ts      # redirects unauthenticated users away from /admin (UX guard)
  pages/                   # file-based routes (public + /admin/**)
server/
  api/public/**            # read-only endpoints for public pages
  api/registrations.post   # public: competition registration (anti-spam)
  api/newsletter.post      # public: newsletter signup (anti-spam)
  api/admin/**             # admin CRUD + auth + upload (all require a session)
  database/schema.ts       # Drizzle tables (single source of truth)
  database/client.ts       # DB connection (driver switch point)
  database/migrations/     # generated SQL migrations
  utils/                   # requireAdmin, rateLimit, validation (Zod), sanitize, crud factory, queries
scripts/seed.ts            # admin + sample data seeder
public/uploads/            # uploaded images (gitignored; keep on the server)
docs/                      # planning + design documents
```

## 10. Admin guide (running a yearly edition)

1. **Events → New event** → set title, year, dates, venue, hero image → **Make current** (exactly one event is the live one on the home page).
2. **Open the event → Add competition** → description, rules, prizes, registration open/close + deadline, team settings.
3. **Home page** sections (left nav group): **Why Join**, **Timeline**, **Sponsors** (upload company logos), **Judges & Speakers**, **Winners**, **FAQ**, **Newsletter**. Each auto-hides on the site when empty.
4. **Settings** → edit hero text, stats, section headings, venue + map, and show/hide any section.
5. **Registrations** → review, confirm/reject, **Export CSV**. **Newsletter** → view subscribers, **Export CSV**.
6. After the event → set its status to **Past** and add **Gallery** photos; it then shows in the public archive.

## 11. Troubleshooting

| Symptom | Fix |
|---|---|
| `npm run build` fails with **"Cannot find native binding"** (rolldown, Windows) | npm optional-dependency bug. Run `npm install --no-save @rolldown/binding-win32-x64-msvc@<rolldown version>` (get the version with `node -e "console.log(require('rolldown/package.json').version)"`), then build again. |
| Build fails citing **`.nuxt`** / file in use | Stop the dev server first; don't run `dev` and `build` at the same time (they share `.nuxt`). |
| `db:migrate` error **"directory does not exist"** | Already handled — the script auto-creates `.data/`. If you call `drizzle-kit migrate` directly, `mkdir .data` first. |
| Port 3000 in use | `npm run dev -- --port 3001`. |
| Admin login fails | Confirm `.env` `ADMIN_EMAIL`/`ADMIN_PASSWORD`, then `npm run seed` to (re)apply them. |
| Native module error on `npm install` (Windows) | Install VS Build Tools "Desktop development with C++", delete `node_modules`, reinstall. |

## 12. Security model (summary)

- Every `/api/admin/**` route calls `requireAdmin()` (session check) before reading input or touching the DB.
- Exactly **two** public write endpoints: `POST /api/registrations` and `POST /api/newsletter` — both rate-limited, honeypot + time-trap protected, Zod-validated, with server-side dedupe/window checks.
- Rich text is sanitized server-side on write; uploaded SVGs are sanitized and served under a locked-down CSP; CSV exports escape formula-leading characters; registration PII is never exposed in public payloads.

Full detail: [`docs/Security_Plan.md`](docs/Security_Plan.md).
